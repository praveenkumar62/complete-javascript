JavaScript Basics

Var vs Let vs Const

Var

  • Values can be reassigned
  • (Function scope) If declared in a function, then accessed inside a function. If declared globally, then can be accessed from anywhere

Let

  • Values can be reassigned
  • (Function scope) Once it declared inside a scope, cant be used outside of that scope

Const

  • Values cannot be reassigned
  • (Function scope) Once it declared inside a scope, cant be used outside of that scope

Important to know about Values and References

In Primitive Data types, variables are copied by value

  • String
  • Number
  • Boolean
  • Symbol
  • Undefined
  • Null

In Non-Primitive Data types, variables are copied by the reference

  • Object
  • Function
  • Arrays

Copying by Value

Example 1:

            var x = 10;
            var y = x;
            x = 20;
            // x has 20
            // y has 10

Example 2:

            var x = 10;
            function calling(n) {
                n++;
            }
            calling(x);
            // now result of x is 10 still
            //This is because value 10 is stored inside a function, and not on x.

This above variable copies by value

Copying by Reference

Example 1:

            var x = {val: 10};
            var y = x;
            x.val = 20;
            // x has {val: 20}
            // y has {val: 20}
            //This is because value is copied by reference [It points the memory not the value, so value gets changed]

Example 2:

            var x = {val: 10};
            function calling(n) {
                value.n++;
            }
            calling(x);
            // result of x is {val: 10} before function call
            // now result of x is{val: 11} after function call

REST Operator

If we dont know how much arguments we are getting in a function, then we use REST operator to get

            function testing(target, ...vals) {
                console.log('target is' + target);
                console.log(...vals);
            }
            testing(1, 2, 3, 4, 5);     
            // First argument in function definition is target, so it takes arg 1, which is a first argument in function calling
            // If we use target2 as second argument, then in function call, second argument is taken for that
            // Remaining arguments in function calling are taken inside the REST operator, which gives as an array in the
            // function definition (...vals), that can be used if we use SPREAD operator inside functions.

Loops for Objects & Arrays - For in & For of

Example 1: For In Loops - Used for enumerating through Objects

            var datas = {
                name: 'CP',
                age: 22,
                place: In,
                draw: function() {
                    console.log('drawing...');
                }
            }
            for (let item in datas) {
                console.log('Property - ' + item + ', Values - ' + datas[item] );
            }
            // OUTPUT: Property - name, Values - CP
            //        Property - age, Values - 22
            //        Property - place, Values - In
            //        Property - draw, Values - function() { console.log('drawing...'); }

Example 1: For Of Loops - Used for enumerating through Arrays

            var arrs = [1, 2, 3, 4];
            for (let item of arrs) {
                console.log(item);
            }
            // OUTPUT: 1
            //         2
            //         3
            //         4

Predefined methods for finding an element

If we need to find element for primitive types, use

  • indexOf()
  • lastIndexOf()
  • search

Example: 1

            let arr = [1, 2, 3, 4];
            arr.indexOf(2);
            // 1

If we need to find element for reference types, use

  • find()
  • findIndex()

Example: 1 - find() method

            let arr3 = [20, 21, 22, 23, 24];
            let arrGot1 = arr3.findIndex(function(e) {
                return e == 22;
            });
            console.log(arrGot1);
            // 22

Example: 2 - findIndex() method

            let arr4 = [
                {n:1 , age: 20}, {n:2, age: 22}
            ];
            let arrGot2 = arr4.findIndex(function(e) {
                return e.age == 22;
            });
            console.log(arrGot2);
            // {n:2, age: 22}

Hoisting

It is a process of bringing the function declarations to the top is called Hoisting.

JavaScript automatically does hoisting

Only works for FUNCTION DECLARATION, NOT for Function expression

Real example for HOISTING

            // Function Declaration
            work();
            function work() {
                console.log('working...');
            }

            // Function Expression
            run();      // ERROR, since it is called before declaring
            let run = function() {
                console.log('running....');
            }

Factory Functions [OOPS]

Traditional way

            var obj1 = {
                name: "Praveen",
                age: 22,
                address: {
                    continent: "Asia",
                    country: "India"
                },
                work: function() {
                    console.log('Start');
                }
            }

Modern way - If using same structure to create multiple objects, using functions is easy

            function Creation(name, age) { 
                return {
                    name,       // also as name: name
                    age,        // also as age: age
                    address: {
                        country: "india"
                    },
                    work() {
                        console.log('Start');
                    }	
                }	
            } 
            var obj2 = Creation("Kumar", 20);

Constructor Functions [OOPS]

Cloning an Object

Method - 1

            var origObj = {
                name: "Sam",
                age: 19
            }
            let cloneObj = Object.assign({}, origObj);   // {} - this creates a empty object and clones inside it

            // Cloning a object to exisiting object
            var existObj = Object.assign({
                experience: 2
            }, origObj);

Method - 2 [ Using Spread Operator ]

            let origObj1 = {
                name: "Raj",
                age: 19
            }
            let cloneObj1 = { ...origObj1 };   // This creates a copy to new object

            clon1 = { ...origObj1, exp: 2 };   // We can add, modify the cloning object by adding to it
